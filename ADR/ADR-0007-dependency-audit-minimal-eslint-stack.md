# ADR-0007: Dependency audit и минимальный ESLint stack

## Статус

Принято

## Контекст

Задача `FND-007` требует минимизировать dependency tree и удалить неиспользуемые/дублирующие пакеты без ухудшения quality gates.

После ревизии workspace зависимостей выявлено, что `@eslint/js` больше не обязателен:

- ранее использовался только для `js.configs.recommended`;
- текущие проверяемые файлы ограничены baseline JavaScript-кодом сервисов;
- необходимые правила можно явно задать в локальной конфигурации.

## Решение

1. Удалить root devDependency `@eslint/js`.
2. Обновить `eslint.config.mjs`:
   - убрать импорт `@eslint/js`;
   - задать минимально необходимые правила явно:
     - `no-undef`
     - `no-unused-vars` (с `argsIgnorePattern: "^_"`)
3. Обновить lockfile.

## Последствия

- Плюсы:
  - уменьшен набор зависимостей root workspace;
  - упрощена поддержка линт-конфига без внешнего preset-пакета;
  - quality gates остаются зелеными.
- Минусы:
  - часть “recommended” правил ESLint теперь не подключена автоматически;
  - при усложнении кодовой базы набор правил нужно будет расширять осознанно.
