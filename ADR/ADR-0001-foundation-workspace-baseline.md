# ADR-0001: Foundation workspace baseline для Stage 1

## Статус

Принято

## Контекст

Первая задача `FND-001` требует быстро подготовить единообразный каркас для четырех сервисов (`api`, `ui`, `bot`, `scheduler`) с общими правилами baseline и командами валидации (`install`, `build`, `test`).

На старте в репозитории отсутствовал код сервисов и инфраструктурный baseline.

## Решение

1. Выбран единый workspace-подход на `pnpm` с пакетами:
   - `api`
   - `ui`
   - `bot`
   - `scheduler`
2. Зафиксированы требования среды:
   - `Node >=20`
   - `pnpm >=10`
3. Для каждого сервиса создан одинаковый минимальный baseline:
   - `README.md`
   - `LICENSE` (MIT)
   - `.editorconfig`
   - `.env.example`
   - `package.json` с командами `build`, `test`, `dev`
   - `src/index.js` placeholder entrypoint
4. Добавлены root-команды для запуска сразу всех пакетов (`build`, `test`, `dev`) и инструкция по локальному старту.

## Последствия

- Плюсы:
  - единая структура для всех сервисов;
  - быстрый onboarding и предсказуемый DX;
  - можно постепенно наполнять сервисы реальной логикой без реорганизации структуры.
- Минусы:
  - на этом шаге `build/test` носят bootstrap-характер и не проверяют бизнес-логику.
