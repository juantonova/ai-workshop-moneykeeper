# ADR-0006: Cleanup policy для bootstrap-артефактов

## Статус

Принято

## Контекст

Задача `FND-006` требует удалить временные bootstrap-файлы/заглушки, оставшиеся после инициализации, и оставить только реально используемые артефакты.

На момент cleanup в репозитории оставались:

- демонстрационные stub-файлы, созданные для начального scaffolding;
- сгенерированные артефакты (`contracts/dist`, `contracts/src/generated`, `api/openapi/openapi.json`), закоммиченные в историю.

## Решение

1. Удалены bootstrap sample stubs в `api/ui/bot/scheduler`.
2. Сервисные entrypoint-файлы упрощены до минимального запуска без demo-импортов.
3. Сгенерированные артефакты больше не хранятся в git:
   - добавлены ignore-правила в `.gitignore`;
   - артефакты удалены из индекса.
4. Генерация артефактов оставлена в build-пайплайне:
   - `api` генерирует OpenAPI;
   - `contracts` генерирует client + собирает `dist`.

## Последствия

- Плюсы:
  - репозиторий чище, меньше bootstrap-мусора;
  - меньше noise в diff при регенерации контрактов;
  - остаются только файлы, которые действительно являются исходниками.
- Минусы:
  - локально generated-артефакты появляются только после build/generate шагов.
