# API-001 — Initial Prisma schema for core domain

## Что сделано

- Подключен Prisma-стек в `api`:
  - зависимости `prisma`, `@prisma/client`;
  - конфигурация `api/prisma.config.ts` и datasource `SQLite`.
- Реализована core-схема в `api/prisma/schema.prisma`:
  - модели `User`, `RefreshSession`, `Account`, `Category`, `Transaction`;
  - enum-типы для валют, типов счетов/категорий/транзакций и статусов;
  - связи между сущностями и индексы под основные CRUD/history сценарии.
- Сгенерирована и применена первая миграция:
  - `api/prisma/migrations/20260226104644_init_core_domain/migration.sql`.
- Обновлены API-скрипты:
  - `prisma:generate`, `prisma:migrate:dev`, `prisma:migrate:deploy`;
  - `build` расширен генерацией Prisma Client.
- Обновлен пример env:
  - `api/.env.example` дополнен `DATABASE_URL`.
- Задача помечена выполненной в `BACKLOG.md`.

## Верификация

- `pnpm --filter @moneykeeper/api run quality`
- `pnpm quality`

Результат: все quality gates workspace проходят (lint/format/test/build), миграция применена локально, Prisma Client генерируется успешно.
