# FND-002 — Establish shared engineering conventions

## Что сделано

- Во всех сервисах (`api`, `ui`, `bot`, `scheduler`) подключены одинаковые quality gates:
  - `lint`
  - `format`
  - `format:check`
  - `test`
  - `build`
  - `quality`
- На уровне workspace добавлены агрегированные команды качества.
- Подключены root-инструменты и конфиги:
  - `eslint.config.mjs`
  - `.prettierrc.json`
  - `.prettierignore`
- Добавлен `CONVENTIONS.md` с едиными правилами:
  - линтинг/форматирование
  - commit convention (Conventional Commits)
  - branch naming
  - naming conventions по сервисам
- Naming conventions применены в кодовых примерах:
  - API DTO: `api/src/dto/create-transaction.dto.js`
  - UI component: `ui/src/components/TransactionForm.js`
  - Bot handler: `bot/src/handlers/create-expense.handler.js`
  - Scheduler job: `scheduler/src/jobs/rebuild-monthly-snapshot.job.js`

## Верификация

- `pnpm install`
- `pnpm quality`

Ожидаемый результат: quality gates проходят для всех сервисов.
